# Validation

Ventyd is built on the [Standard Schema](https://standardschema.dev) specification, which provides a unified interface for validation libraries. This means you can choose your favorite validation library and use it with Ventyd seamlessly.

## Standard Schema Overview

Standard Schema is an emerging standard that allows different validation libraries to work with the same code. Instead of being locked into a single library's ecosystem, you can:

- **Choose your preferred library** based on your needs
- **Switch libraries** without major refactoring
- **Use community-built integrations** across different tools
- **Maintain type safety** throughout your application

All official Ventyd integrations automatically handle:
- Event namespacing (e.g., `user:created`, `product/updated`)
- Event metadata (eventId, eventName, eventCreatedAt, etc.)
- Type inference for both events and state
- Validation of incoming data

## Quick Selection Guide

### Choose Valibot if:
- You want the smallest bundle size
- You prefer modular, tree-shakeable code
- You're building a performance-critical application
- You like composable, readable schema definitions

### Choose Zod if:
- You want the largest, most mature ecosystem
- You need extensive documentation and community support
- You prefer familiar, chainable API
- You want lots of existing integrations

### Choose TypeBox if:
- You need JSON Schema compatibility
- You're building APIs or need OpenAPI integration
- You want the fastest validation performance
- You need runtime type checking with JSON Schema

### Choose ArkType if:
- You want ultra-fast validation (10x faster than Zod)
- You prefer concise, elegant syntax
- You want excellent error messages
- You're validating complex nested structures frequently

## Quick Example

Here's how to use each library with Ventyd:


  
    ```typescript
    import { defineSchema } from 'ventyd';
    import { valibot, v } from 'ventyd/valibot';

    const userSchema = defineSchema("user", {
      schema: valibot({
        event: {
          created: v.object({
            email: v.pipe(v.string(), v.email()),
            name: v.string()
          })
        },
        state: v.object({
          email: v.string(),
          name: v.string()
        })
      }),
      initialEventName: "user:created"
    });
    ```
  

  
    ```typescript
    import { defineSchema } from 'ventyd';
    import { zod, z } from 'ventyd/zod';

    const userSchema = defineSchema("user", {
      schema: zod({
        event: {
          created: z.object({
            email: z.string().email(),
            name: z.string()
          })
        },
        state: z.object({
          email: z.string(),
          name: z.string()
        })
      }),
      initialEventName: "user:created"
    });
    ```
  

  
    ```typescript
    import { defineSchema } from 'ventyd';
    import { typebox, Type } from 'ventyd/typebox';

    const userSchema = defineSchema("user", {
      schema: typebox({
        event: {
          created: Type.Object({
            email: Type.String({ format: 'email' }),
            name: Type.String()
          })
        },
        state: Type.Object({
          email: Type.String(),
          name: Type.String()
        })
      }),
      initialEventName: "user:created"
    });
    ```
  

  
    ```typescript
    import { defineSchema } from 'ventyd';
    import { arktype, type } from 'ventyd/arktype';

    const userSchema = defineSchema("user", {
      schema: arktype({
        event: {
          created: type({
            email: "string.email",
            name: "string"
          })
        },
        state: type({
          email: "string",
          name: "string"
        })
      }),
      initialEventName: "user:created"
    });
    ```
  


## Custom Namespace Separators

All official integrations support custom namespace separators. By default, events are named with a colon separator (e.g., `user:created`), but you can customize this:

```typescript
const userSchema = defineSchema("user", {
  schema: valibot({
    event: {
      created: v.object({ ... }),
      updated: v.object({ ... })
    },
    state: v.object({ ... }),
    namespaceSeparator: "/" // Events become "user/created", "user/updated"
  }),
  initialEventName: "user/created"
});
```

<Callout type="info">
**Available Separators**

You can use any string as a separator:
- `":"` (default) - `user:created`
- `"/"` - `user/created`
- `"."` - `user.created`
- `"-"` - `user-created`
- `"_"` - `user_created`

Just make sure your `initialEventName` matches the separator you choose!
</Callout>

## Getting Started

Ready to pick a library? Here's what to do:

1. **Install Ventyd**: See [Installation Guide](/docs/getting-started/installation)
2. **Choose a library**: Start with [Valibot](/docs/validation/valibot) or browse all options
3. **Define your schema**: Use the library's API to define events and state
4. **Build your entity**: Implement business logic with mutations
5. **Start coding**: Create, update, and query your event-sourced entities

Each validation library guide includes:
- Complete installation instructions
- Basic usage examples
- Advanced patterns and tips
- Library-specific features
- Pros and cons comparison
- Custom namespace separator examples

<Callout type="warn">
**Official Integrations vs Standard Schema**

Official integrations like `ventyd/valibot`, `ventyd/zod`, `ventyd/typebox`, and `ventyd/arktype` automatically handle event namespacing and metadata. If you use the `standard()` provider directly, you must manually include all event metadata fields in your schemas. See [Standard Schema Provider](/docs/validation/standard-schema) for details.
</Callout>

## Want More Libraries?
